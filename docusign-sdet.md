# 🔍 1. What this role actually is (important)

This is **NOT a pure automation tester role**.

It is a mix of:

* **SDET (Automation + Framework building)**
* **Data QA / Data Engineer testing**
* **Low-code automation (UiPath)**

👉 So you need **3 strong pillars**:

1. Automation Engineering
2. Data Testing
3. System / API Understanding

---

# 🧠 2. Technologies You MUST Know (with level)

## ✅ A. Automation Testing (VERY IMPORTANT)

**Level expected: Strong (Interview focus area)**

### Must know:

* Python (preferred) OR Java
* Selenium / Playwright
* Test frameworks (PyTest / TestNG / JUnit)

### What you should be able to do:

* Build a **framework from scratch**
* Explain:

  * Page Object Model (POM)
  * Data-driven testing
  * Parallel execution
* Write automation for:

  * UI + API

👉 **Interview questions:**

* How do you design an automation framework?
* How do you handle flaky tests?
* Difference: Selenium vs Playwright?
* How do you run tests in CI/CD?

---

## ✅ B. API Testing (HIGH PRIORITY)

**Level expected: Strong**

### Must know:

* REST APIs
* JSON, XML
* Tools: Postman / RestAssured / Python requests

### You should be able to:

* Validate API responses
* Write automated API tests
* Handle auth (tokens, OAuth basics)

👉 **Interview questions:**

* What is REST?
* Difference between PUT vs POST?
* How do you validate API response?
* How do you automate API testing?

---

## ✅ C. Data Testing (VERY IMPORTANT ⭐)

This is the **core differentiator** in this JD.

**Level expected: Strong (many candidates fail here)**

### Must know:

* SQL (Advanced)
* ETL testing concepts
* Data warehouse basics

### Tools mentioned:

* Snowflake
* Tableau (basic understanding is enough)

### You should be able to:

* Write complex SQL:

  * Joins
  * Aggregations
  * Window functions (bonus)
* Perform:

  * Source → Target validation
  * Data reconciliation
  * Data quality checks

👉 **Interview questions:**

* What is ETL testing?
* How do you validate data pipelines?
* Write SQL to compare two tables
* What are data quality checks?
